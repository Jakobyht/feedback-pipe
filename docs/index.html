● Step-by-step setup guide

Turn app feedback into agent work.

User feedback becomes a GitHub issue, and a small watcher on your own machine hands that issue to your coding agent. You wire it up once — this page shows every step.

Start the setup ↓ GitHub project ↗
How it works

Three moving parts

Your app only saves feedback. Supabase turns it into a GitHub issue. A watcher on your machine sees the issue and starts the agent. GitHub in the middle keeps everything visible and reviewable.

Your app

Saves feedback

Inserts one row into a Supabase table.

→
Supabase → GitHub

Creates an issue

An Edge Function opens a labeled GitHub issue.

→
Your machine

Runs the agent

A watcher sees the issue and starts your agent.

Before you start

What you need

Supabase project

Hosts the feedback table, the webhook, and the Edge Function.

GitHub repo + token

Where issues are created. Token needs Issues: read & write.

A machine to run the watcher

Your laptop or a Mac mini with Node.js 20+ and your coding agent installed.

The tutorial

Step-by-step setup

Do these in order. Steps 1–4 set up Supabase → GitHub. Step 5 is the one line your app adds. Step 6 starts the watcher on your machine.

Part 1 Supabase → GitHub issues

  1. Create the feedback table

    Supabase → SQL Editor

    Copy this SQL, paste it into the Supabase SQL Editor, and click Run. It creates the feedback table your app writes to, and locks it down so users can insert feedback but never read it back.

    create table if not exists public.feedback (
  id uuid primary key default gen_random_uuid(),
  message text not null,
  page_url text,
  user_email text,
  github_issue_url text,
  status text not null default 'new',
  created_at timestamptz not null default now()
);

alter table public.feedback enable row level security;

create policy "Anyone can submit feedback"
on public.feedback
for insert
to anon, authenticated
with check (true);

create policy "Users cannot read feedback"
on public.feedback
for select
to anon, authenticated
using (false);

  2. Deploy the Edge Function

    Terminal (Supabase CLI)

    This function receives a new feedback row and creates the GitHub issue. Its code is in supabase/functions/create-github-issue/index.ts.

    supabase functions deploy create-github-issue --no-verify-jwt

  3. Set the function's secrets

    Terminal (Supabase CLI)

    Give the function your GitHub details. Replace the values with your own.

    supabase secrets set GITHUB_OWNER="your-github-user-or-org"
supabase secrets set GITHUB_REPO="your-repo-name"
supabase secrets set GITHUB_TOKEN="your-github-token"
supabase secrets set WEBHOOK_SECRET="make-a-long-random-secret"
    • The GitHub token stays in Supabase — it never touches your app or browser.
    • New issues are labeled user-feedback and agent-ready.

  4. Connect the table to the function

    Supabase → Database → Webhooks

    Create a webhook so every new feedback row calls the function.

    • Table feedback, event INSERT, method POST
    • URL https://YOUR_PROJECT_REF.supabase.co/functions/v1/create-github-issue
    • Header x-feedback-webhook-secret = your WEBHOOK_SECRET
Part 2 Your app

  1. Save feedback from your app

    Your feedback form handler

    This is the only code your app adds. Put it where the user's feedback is submitted — it just inserts a row. Supabase and GitHub do the rest.

    await supabase.from("feedback").insert({
  message: feedbackMessage,
  page_url: window.location.pathname,
  user_email: user?.email ?? null
});
Part 3 Your machine

  1. Start the watcher

    Terminal on your laptop / Mac mini

    It polls GitHub for issues labeled agent-ready and starts your agent on the repo. Leave the window open — closing it stops the watcher.

    GITHUB_OWNER="your-github-user-or-org" \
GITHUB_REPO="your-repo-name" \
GITHUB_TOKEN="your-github-token" \
AGENT_REPO="/absolute/path/to/the/code/repo" \
npm run github-agent

    The agent defaults to Claude Code. Override it with AGENT_COMMAND to use Codex, Aider, or anything else.

Verify

Test the whole flow

Insert one feedback row. It should become a GitHub issue, and the watcher should start the agent.

insert into public.feedback (message, page_url, user_email)
values ('The checkout button does nothing on mobile', '/cart', 'test@example.com');
Expected: a new GitHub issue labeled agent-ready appears, and the watcher terminal shows it starting the agent.
No issue appears

Check the webhook URL, the secret header, and the token's Issues permission.

Issue but no agent

Confirm the issue has agent-ready and the watcher is running.

Runs only once

Intended — handled issues are remembered in .ape/github-agent/processed.json.

Integration

Add it to your app

The whole app-side change is inserting one row into Supabase when feedback is submitted. Pick the version closest to your stack.

Never put the GitHub token in your app. It lives only in the Supabase function's secrets. Your app talks to Supabase, nothing else.
