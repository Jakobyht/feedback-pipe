Feedback Pipe
user feedback → your agent

A tiny local program that forwards a user's feedback straight to your own coding agent — Claude Code, Codex, Aider, anything. No model, no judgement. It just carries the message and gets out of the way.

Quick start View on GitHub ↗

The exact process

Everything that happens, end to end — and the one spot where your code fits.

1

A user sends feedback in your app

They type into the feedback form, widget, or support box you already have. Nothing changes for them.

2

Your app calls the pipe ← your code goes here

Inside the handler that already runs when feedback is submitted, you add one HTTP POST to the pipe, carrying your key. That single call is the only code you write. (See the snippet for your language below.)

3

The pipe forwards it

It writes the user's words verbatim into a prompt file in your repo and launches your agent there.

4

Your agent fixes the code

Claude Code (or Codex / Aider) reads the feedback and edits the repo. The pipe does not wait or report back.

01Install

Requires Node.js 20+ and your agent installed (Claude Code by default).

git clone https://github.com/Jakobyht/feedback-pipe.git
cd feedback-pipe
npm install

02Configure

Copy the template to .env (per-machine, never committed) and set two values.

cp .env.example .env
VariableRequiredMeaning
PIPE_API_KEYyesSecret callers must send. Any string — use a long random one.
PIPE_REPOyesAbsolute path to the repo the agent works on.
PIPE_AGENT_COMMANDnoAgent to run. Defaults to Claude Code headless.
PIPE_PORTnoListen port (default 8181).
PIPE_HOSTnoBind address (default 127.0.0.1; use 0.0.0.0 for remote calls).

03Run

That's the only command needed to keep it running.

npm run pipe

The API — and how to test it

One endpoint. POST your key and a JSON body — only message is required.

POST http://<host>:8181/feedback
Authorization: Bearer <PIPE_API_KEY>
Content-Type: application/json

{ "message": "The checkout button does nothing on mobile", "pageUrl": "/cart" }

Responds 202 { "taskId": "...", "status": "forwarded" }. The pipe does not wait for the agent or report back.

Test it first with curl. Run this in a terminal to confirm the pipe works before touching your app. This curl is only a test — you do not put it in your code.
curl -sS http://localhost:8181/feedback \
  -H "authorization: Bearer YOUR_PIPE_API_KEY" \
  -H 'content-type: application/json' \
  -d '{"message":"The checkout button does nothing on mobile","pageUrl":"/cart"}'

Where this goes in your app

Once curl works, drop the same call into your real code.

Where exactly? Find the function in your app that already runs when a user submits feedback — your form handler, your /feedback route, your support endpoint. Right where you save or log that message, add the call below. That is the whole integration: one request, in a place that already exists.
Keep the key safe: call the pipe from your backend so PIPE_API_KEY stays server-side. From a browser, the key is visible to users — front the pipe with your own server instead.
